# BanaPE UI 设计改进指南

## 设计原则

### 1. 步骤导向，流程清晰
- ✅ 三步流程：选择源 → 配置组件 → 构建
- ✅ 顶部步骤指示器，当前步骤高亮
- ✅ 用户始终知道自己在哪里，下一步是什么

### 2. 操作简洁，路径最短
- ✅ 主要操作大卡片设计，一键点击
- ✅ 常用操作直接展示，高级选项折叠
- ✅ 减少输入，多用选择和默认值

### 3. 信息层次分明
- ✅ 重要信息突出显示（WIM检测结果）
- ✅ 次要信息折叠（ADK路径）
- ✅ 状态实时反馈（加载中、就绪）

### 4. 反馈及时明确
- ✅ 加载状态动画提示
- ✅ 成功/错误状态明确
- ✅ 引导用户下一步操作

## 界面元素规范

### 步骤指示器
```
[1] ───── [2] ───── [3]
选择源     配置组件    构建
```
- 当前步骤：主色圆形 + 白色数字
- 已完成：绿色 ✓
- 未到达：灰色

### 主操作卡片
- 大图标 (32px)
- 标题清晰
- 描述/当前值
- 右箭头提示可点击
- Hover 效果：边框高亮 + 微上移

### 信息展示
- 网格布局，自适应
- 标签小号大写 + 值大号加粗
- 成功信息左侧绿色边框

### 状态提示
- 底部状态栏
- 图标 + 文字
- 颜色区分状态（绿=成功，蓝=加载，红=错误）

## 颜色规范

### 语义色
- **主色** (Primary): `#4A90D9` - 按钮、链接、高亮
- **成功** (Success): `#4CAF50` - 完成、就绪
- **警告** (Warning): `#FF9800` - 注意、提示
- **错误** (Error): `#F44336` - 失败、错误
- **信息** (Info): `#2196F3` - 提示、引导

### 中性色
- **文字主色**: `#333333` (亮) / `#E0E0E0` (暗)
- **文字次色**: `#666666` (亮) / `#A0A0A0` (暗)
- **背景主色**: `#FFFFFF` (亮) / `#1E1E1E` (暗)
- **背景次色**: `#F5F5F5` (亮) / `#2D2D2D` (暗)
- **边框**: `#E0E0E0` (亮) / `#404040` (暗)

## 间距规范

- **超小**: 4px - 图标和文字间距
- **小**: 8px - 相关元素间距
- **中**: 16px - 卡片内边距
- **大**: 24px - 区块间距
- **超大**: 32px - 页面边距

## 字体规范

- **标题**: 18px, 600 (粗体)
- **副标题**: 16px, 600
- **正文**: 14px, 400
- **小字**: 12px, 400
- **标签**: 12px, 500, 大写

## 交互规范

### 按钮
- 主按钮：主色背景 + 白字
- 次按钮：透明背景 + 主色边框
- 图标按钮：透明背景 + Hover 变色
- Disabled：50% 透明度

### 输入框
- 默认：浅灰边框
- Focus：主色边框 + 微阴影
- Error：红色边框 + 错误提示

### 卡片
- 默认：浅灰背景 + 圆角
- Hover：边框高亮 + 微上移 + 阴影
- Active：边框主色
